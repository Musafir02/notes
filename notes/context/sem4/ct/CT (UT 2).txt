**Context Free Grammar (CFG)**

A Context Free Grammar (CFG) is a type of grammar that defines the structure of a language using production rules. These rules specify how to generate strings of symbols from a set of non-terminal symbols. A CFG consists of:

*   A set of terminal symbols (e.g., letters, numbers, punctuation marks)
*   A set of non-terminal symbols (e.g., keywords, identifiers)
*   A set of production rules that specify how to replace non-terminal symbols with sequences of terminal and non-terminal symbols

For example, consider the following CFG:

S → AB | CD

Here, S is the start symbol, and A, B, C, and D are non-terminal symbols. The production rule specifies that S can be replaced by either AB or CD.

**Compiler Design**

Compiler design is the process of creating a compiler, which is a program that translates source code written in a high-level programming language into machine code that can be executed by a computer. A compiler consists of several stages:

1.  **Lexical Analysis**: The first stage of compilation, where the source code is broken down into individual tokens (e.g., keywords, identifiers, literals).
2.  **Syntax Analysis**: The second stage, where the tokens are analyzed to ensure they conform to the syntax rules of the programming language.
3.  **Semantic Analysis**: The third stage, where the syntax tree is analyzed to ensure that the program is semantically correct.
4.  **Intermediate Code Generation**: The fourth stage, where the syntax tree is translated into intermediate code (e.g., assembly code).
5.  **Optimization**: The fifth stage, where the intermediate code is optimized for performance.
6.  **Code Generation**: The final stage, where the optimized intermediate code is translated into machine code.

**Turing Machine**

A Turing Machine is a mathematical model of computation that consists of:

*   A finite state machine (FSM) that can read and write symbols on an infinite tape.
*   A set of states (e.g., start state, accept state, reject state).
*   A set of transition rules that specify how to move the head of the tape and change the state based on the current symbol.

For example, consider the following Turing Machine:

M = (Q, Σ, Γ, δ, q0, F)

**Decidability Problems**

A decidability problem is a problem that can be solved by a Turing Machine in a finite number of steps. Decidability problems are typically classified into two categories:

*   **Decidable**: Problems that can be solved by a Turing Machine in a finite number of steps.
