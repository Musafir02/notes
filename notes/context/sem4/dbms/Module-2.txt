**Database Management System Notes**

### Main Topic: Relational Model Constraints

The page focuses on the importance of integrity constraints in maintaining data integrity within a relational database management system (DBMS). These constraints enforce limits on the data or restrictions on data that can be inserted, updated, or deleted from a table.

### Types of Integrity Constraints

There are three main types of integrity constraints:

*   **Domain Constraints**: These constraints specify the valid values for a particular attribute in a table. For example, a constraint might ensure that all employee salaries are between 0 and 100,000.
*   **Key Constraints**: These constraints define the primary key of a table, which uniquely identifies each row. A primary key can be a single column or a combination of columns.
*   **Entity Integrity Constraints**: These constraints ensure that every child row in a table has a valid parent row in another table. This prevents orphan records from being created.
*   **Referential Integrity Constraints**: These constraints ensure that every child row in a table has a valid parent row in another table. This prevents broken links between tables.

### Need for Integrity Constraints

Integrity constraints are essential in DBMS to maintain accurate, consistent, and trustworthy data over time. They prevent invalid data from entering the database, enforce business rules, improve reliability and trust in queries/reports, reduce manual data cleaning and bugs, and provide data accuracy.

### Key Features of Integrity Constraints in DBMS

Integrity constraints in DBMS have several key features:

*   **Data Accuracy**: Integrity constraints ensure that only correct and proper data is added to the database.
*   **Consistency Enforcement**: They keep related tables consistent by setting rules that make sure foreign key values match the correct primary key values in the linked tables.
*   **Automatic Validation**: The DBMS runs all integrity checks by itself, making sure every INSERT, UPDATE, or DELETE action follows all the rules and stops actions that break them.
*   **Prevention of Invalid Data**: Integrity constraints prevent primary keys, blank values in needed fields, or values not allowed from being inserted or updated.

### Summary

In summary, integrity constraints are crucial in maintaining data integrity within a relational DBMS. They enforce limits on data, define the primary key, and ensure that every child row has a valid parent row. By preventing invalid data from entering the database, enforcing business rules, improving reliability and trust, reducing manual data cleaning and bugs, and providing data accuracy, integrity constraints play a vital role in ensuring the quality and trustworthiness of data in a DBMS.

**Properties of Relational Algebra**

The page begins by highlighting four essential properties of relational algebra: closure, idempotence, commutativity, and associativity. These properties ensure that the operations performed on the relational algebra are consistent and predictable, making it easier to manage and analyze data.

*   **Closure Property**: This property states that every operation takes one or more relations as input and produces another relation as output, allowing operations to be nested or chained without breaking the relational structure.
*   **Idempotence**: Applying the same operation multiple times yields the same result as applying it once. This means that repeating an operation does not change the outcome.
*   **Commutativity**: Certain operations are order-independent, meaning that the order in which they are performed does not affect the result. For example, union (R ∪ S = S ∪ R) and intersection (R ∩ S = S ∩ R) operations can be commuted.
*   **Associativity**: Binary set operations like union and intersection associate, meaning that (R ∪ S) ∪ T = R ∪ (S ∪ T), enabling flexible grouping in expressions.
